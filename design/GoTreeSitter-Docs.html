gotreesitter

Tree-sitter is a parser generator tool and an incremental parsing library. gotreesitter is its runtime, reimplemented in pure Go — the same parse-table format, the same .scm queries, ABI 15, no C toolchain.

Getting Started

From zero to a
syntax tree.

One go get. No compiler, no CGo, no grammar sources to vendor. Here's the whole path.

1

Install

Pure Go — it resolves like any other module. Nothing to build.

shell
go get github.com/odvcencio/gotreesitter
2

Grab a language and parse

206 grammars ship in the grammars registry. Pull one, make a parser, parse bytes.

main.gogo
package main

import (
    "fmt"
    "github.com/odvcencio/gotreesitter"
    "github.com/odvcencio/gotreesitter/grammars"
)

func main() {
    src := []byte("package main\n\nfunc main() {}\n")

    lang   := grammars.GoLanguage()
    parser := gotreesitter.NewParser(lang)

    tree, _ := parser.Parse(src)
    fmt.Println(tree.RootNode())
}
3

Or detect the language from a filename

DetectLanguage resolves a path to the right LangEntry — extensions, shebangs, and markdown fence aliases all wired in.

detect.gogo
entry := grammars.DetectLanguage("main.go")
lang  := entry.Language()
4

Decide how strict you want to be

The default methods keep tree-sitter's partial-tree behavior: a timeout, cancellation, or safety limit stops the parse but still returns a usable tree, error nil. Reach for the Strict variants when a partial tree should fail the request.

strict.gogo
parser.SetTimeoutMicros(50_000)

tree, err := parser.ParseStrict(src)
if errors.Is(err, gotreesitter.ErrParseStoppedEarly) {
    fmt.Println(tree.ParseStopReason())
}
Good to know

Strict variants exist for every entry point — full parse, incremental, token-source, factory, ParseWith, and ParserPool. Partial trees always record their stop reason via tree.ParseStopReason().

Guide

Parsing
& trees.

A parse returns a Tree. Everything else — walking, lookup, extraction — hangs off its nodes.

Trees & nodes

Print any node as an S-expression, or turn an editor byte offset straight into the smallest matching node with NodeAtByte / NamedNodeAtByte — no hand-rolled tree walk, exact end-byte boundaries handled.

lookup.gogo
tree, _ := parser.Parse(src)

// smallest named descendant covering an editor offset
node := tree.NamedNodeAtByte(offset)

// the definition that encloses a cursor position
enclosing, ok := tree.EnclosingDefinition(offset)

The tree cursor

TreeCursor keeps an explicit (node, childIndex) frame stack. Parent, child, and sibling moves are O(1) with zero allocations — sibling traversal indexes straight into the parent's children[] slice.

cursor.gogo
c := gotreesitter.NewTreeCursorFromTree(tree)

c.GotoFirstChild()
c.GotoChildByFieldName("body")

for ok := c.GotoFirstNamedChild(); ok; ok = c.GotoNextNamedSibling() {
    fmt.Printf("%s at %d\n", c.CurrentNodeType(), c.CurrentNode().StartByte())
}

idx := c.GotoFirstChildForByte(128)
Heads up

Cursors hold direct pointers into tree nodes. Recreate them after Tree.Release(), Tree.Edit(...), or an incremental reparse.

Code-understanding, one pass

For hot indexing paths that need common symbols but not arbitrary tags-query semantics, skip the query fanout entirely. These extractors cover common Go, JavaScript, TypeScript/TSX, Python, Starlark and Java shapes and skip anything ambiguous conservatively.

extract.gogo
defs  := gotreesitter.ExtractDefinitionSpans(tree)
calls := gotreesitter.ExtractCalls(tree)
bases := gotreesitter.ExtractHeritage(tree)

// language-neutral import extraction, source- or tree-based
imports, report := gotreesitter.ExtractImportsFromSourceWithReport(src, lang)
Why it's fast

On a real rules_python replay (564 files, 949 imports) the hybrid source extractor matched full-tree extraction exactly in ~3.9 ms — versus ~1.17 s to parse and walk every tree. It reports a fallback recommendation so you only pay for a full parse when the fast scan can't be sure.

Queries
that match.

The full tree-sitter S-expression pattern language — quantifiers, alternation, fields, anchors, every standard predicate. Plus typed Go codegen the C runtime doesn't have.

Compile & stream

query.gogo
q, _ := gotreesitter.NewQuery(`(function_declaration name: (identifier) @fn)`, lang)
cursor := q.Exec(tree.RootNode(), lang, src)

for {
    match, ok := cursor.NextMatch()
    if !ok { break }
    for _, cap := range match.Captures {
        fmt.Println(cap.Node.Text(src))
    }
}

The pattern language

A query is one or more patterns, each an S-expression that matches a shape in the tree. Every construct tree-sitter defines is supported.

(node_type)

Named node. Matches a node by its grammar type. Nest them to match structure: (call (identifier)).

"literal"

Anonymous node. Matches punctuation/keywords by text — "return", "+".

field:

Field. Constrain a child to a named field: (function_declaration name: (identifier)).

!field

Negated field. Match only when a field is absent: (class !type_parameters).

_   (_)

Wildcards. _ matches any named-or-anonymous node; (_) matches any named node.

@capture

Capture. Name a node so you can read it from the match: (identifier) @name.

?   *   +

Quantifiers. Optional, zero-or-more, one-or-more — on nodes or groups.

[ a b ]

Alternation. Match any one of several patterns: [(true) (false)].

( a . b )

Anchor. . forces adjacency — first/last child, or no named nodes between two patterns.

#predicate?

Predicate. Runtime tests on captures — #eq?, #match?, and friends (full list below).

functions.scmquery
; every exported Go function with a body, name captured
(function_declaration
  name: (identifier) @name
  body: (block) @body
  (#match? @name "^[A-Z]"))

; a call to fmt.Println OR fmt.Printf
(call_expression
  function: (selector_expression
    operand: (identifier) @pkg
    field: (field_identifier) @fn)
  (#eq? @pkg "fmt")
  (#any-of? @fn "Println" "Printf"))

Incremental,
injection & rewriting.

The parts where pure Go pulls decisively ahead: reparse in nanoseconds, parse many languages in one document, and rewrite source atomically.

Incremental reparsing

Tell the tree what changed, reparse. ParseIncremental walks the old tree's spine, re-lexes only the invalidated span, and reuses unchanged subtrees by reference — both leaf and non-leaf, driven by pre-goto state stored on interior nodes.

incremental.gogo
tree, _ := parser.Parse(src)

// user types "x" at byte offset 42
tree.Edit(gotreesitter.InputEdit{
    StartByte:   42,
    OldEndByte:  42,
    NewEndByte:  43,
    StartPoint:  gotreesitter.Point{Row: 3, Column: 10},
    OldEndPoint: gotreesitter.Point{Row: 3, Column: 10},
    NewEndPoint: gotreesitter.Point{Row: 3, Column: 11},
})

tree2, _ := parser.ParseIncremental(src, tree)
When no edit has occurred, ParseIncremental detects the nil-edit on a pointer check and returns in ~2.4 ns with zero allocations. A single-byte edit lands in ~649 ns. Native C re-walks in ~102 µs.

Want to see the subtree reuse light up? Fire an edit in the playground →

Multi-language documents

The injection parser orchestrates HTML+JS+CSS, Markdown code fences, and Vue/Svelte templates. Static (#set!) and dynamic (@injection.language) detection, recursive nesting, and incremental reparse with child-tree reuse.

injection.gogo
ip := gotreesitter.NewInjectionParser()
ip.RegisterLanguage("html", htmlLang)
ip.RegisterLanguage("javascript", jsLang)
ip.RegisterLanguage("css", cssLang)
ip.RegisterInjectionQuery("html", injectionQuery)

result, _ := ip.Parse(source, "html")

for _, inj := range result.Injections {
    fmt.Printf("%s: %d ranges\n", inj.Language, len(inj.Ranges))
    // inj.Tree is the child language's parse tree
}

Atomic source rewriting

Collect replace / insert / delete edits against byte ranges, apply them in a single validated pass, and get back []InputEdit records ready for incremental reparse.

rewrite.gogo
rw := gotreesitter.NewRewriter(src)
rw.Replace(funcNameNode, []byte("newName"))
rw.InsertBefore(bodyNode, []byte("// added\n"))
rw.Delete(unusedNode)

newSrc, _ := rw.ApplyToTree(tree)
newTree, _ := parser.ParseIncremental(newSrc, tree)
Editor-grade

Native UTF-16 input. Parse Go-native code units or endian byte buffers with ParseUTF16 / ParseUTF16Bytes. The core keeps a canonical UTF-8 view internally while the tree maps nodes, edits, ranges, highlights, tags and injections back to UTF-16 code-unit coordinates — so editor offsets never need hand-conversion.

Have a working tree-sitter grammar? Bring it in as a library dependency — not a fork. The whole pipeline is public API against an unmodified module.

1 · Get a grammar IR

Option A — import grammar.json. The fully-resolved output of tree-sitter generate (not grammar.js) is the canonical input: no require(), no helpers to interpret.

Option B — write the Go DSL directly. For small DSLs there's no JavaScript at all. The builders live in grammargen: NewGrammar, Define, Str, Pat, Sym, Seq, Choice, Repeat, Optional, Token, Field, Prec…

kvconf.gogo
func Grammar() *grammargen.Grammar {
    g := grammargen.NewGrammar("kvconf")

    g.Define("document", grammargen.Repeat(grammargen.Sym("entry")))
    g.Define("entry", grammargen.Seq(
        grammargen.Field("key", grammargen.Sym("identifier")),
        grammargen.Str("="),
        grammargen.Field("value", grammargen.Sym("_value")),
    ))
    g.Define("identifier", grammargen.Pat(`[A-Za-z_][A-Za-z0-9_]*`))
    g.SetExtras(grammargen.Pat(`\s`), grammargen.Sym("comment"))
    return g
}

2 · Generate & parse

Compile the IR into runtime tables plus a shippable gob+gzip blob in one pass. Generation is milliseconds for small grammars — so you do it once, at build time, and ship the blob.

gen.gogo
lang, blob, err := grammargen.GenerateLanguageAndBlob(kvconf.Grammar())
os.WriteFile("kvconf.bin", blob, 0o644)

tree, _ := gts.NewParser(lang).Parse(src)
fmt.Println(tree.RootNode().SExpr(lang))
// (document (entry (identifier) (string)) (entry (identifier) (number)))

Same from the CLI, including a doctor that validates, generates, runs embedded tests and parses a sample:

shell
go run ./cmd/grammargen emit   -json src/grammar.json -bin kvconf.bin
go run ./cmd/grammargen emit   -json src/grammar.json -go kvconf_grammar.go -pkg kvconf
go run ./cmd/grammargen doctor -json src/grammar.json -sample testdata/example.kvconf

Ship it as its own Go module

Register your language for side effects and everything the registry powers just works — DetectLanguage, markdown fence aliases, AllLanguages() listing.

pawn.gogo
func init() {
    grammars.RegisterExtension(grammars.ExtensionEntry{
        Name:             "pawn",
        Extensions:       []string{".pwn", ".inc"},
        Aliases:          []string{"pawno"},
        GenerateLanguage: Language,
        HighlightQuery:   highlightQuery,
    })
}
Ambiguity-heavy grammar?

Opt into the GSS-forest GLR fast path with g.WantsForest = true (or "gotreesitter": {"wantsForest": true} in grammar.json). It coalesces equivalent stack tops instead of forking. The forest declines to the production parser on any failure, so you can't get a hard crash from it — but you bypass the byte-parity certification built-ins get, so diff your corpus on and off before shipping.

Scale reality

Runtime state IDs are uint32 (COBOL generates ~67k states). The precise external-scanner LR(1) builder has a 20,000-state budget and transparently falls back to LALR past it — not an error. pawnkit's real Pawn parser (6,818 states, 333 symbols, 5 externals) sits comfortably inside every budget. You need a COBOL-class grammar before budgets are your problem.

Blob provenance — treat as policy

A blob is gob+gzip of the Language struct, and gob tolerates field drift silently — a stale blob still loads, then quietly misparses. Blobs are not portable across engine vintages. Regenerate from grammar.json with the exact module version you link, check the JSON in next to the blob, and diff parse output over your corpus as the acceptance test.

Grammar authoring

External
scanners.

The escape hatch for tokens a regular lexer can't recognize. In upstream tree-sitter that's scanner.c; here it's a Go type implementing ExternalScanner. Most grammars that think they need one, don't.

Do you actually need one?

✗ You do NOT — handle in-grammar
  • Keyword vs identifier → declare a word token
  • Precedence / associativity → prec, prec.left/right/dynamic
  • Comments & whitespace → extras
  • Plain strings & numbers → token(seq(...))
  • C-style preprocessors → line-oriented rules
✓ You DO — context-sensitive lexing
  • Indentation blocks (Python, YAML) — indent stack
  • Heredocs (bash, Perl, Ruby)
  • Nested string interpolation (Elixir, JS templates)
  • Delimiter-matched raw strings (Rust r###"…"###)
  • Optional semicolons / newline significance (Go ASI)
  • Zero-width / sentinel tokens (HTML implicit close, dedents)

The Go interface

language.gogo
type ExternalScanner interface {
    Create() any
    Destroy(payload any)
    Serialize(payload any, buf []byte) int
    Deserialize(payload any, buf []byte)
    Scan(payload any, lexer *ExternalLexer, validSymbols []bool) bool
}
Contract that differs from C

Dual numbering. validSymbols[i] is indexed by external token index (position in the grammar's externals array). But SetResultSymbol(sym) takes the language symbol ID — lang.ExternalSymbols[i], not i. Keep two constant sets: token indexes for gating, symbol IDs for results. Resolve symbols from the Language at construction time.

The lexer API

MethodSemantics
Lookahead() runeCurrent rune; 0 at EOF.
Advance(skip bool)Consume one rune. skip=true moves the token start forward and never the end.
MarkEnd()Set token end = current position. The only way to set the end.
SetResultSymbol(sym)The recognized token — a language symbol ID (see contract).
Column() uint320-based column at the cursor.
HasPreviousBytes(s)No C equivalent — guard content tokens exposed too broadly by merged states.

Optional capability interfaces

SupportsIncrementalReuse()

Declare true only if reusing subtrees across edits is safe. Stateless: yes. Indent stacks: leave unimplemented.

PreservesStateOnScanFailure()

True if a false Scan never mutated the payload — lets the runtime skip defensive state snapshots.

Hard-learned contracts

  • Emit extras; never skip silently. If whitespace is a declared extra, emit it — don't swallow it. C shifts whitespace extras, which advances the parse position; skipping it drifts your recovery anchor one byte off C's.
  • EOF mirrors C. At end of input with nothing matched, return the zero-Symbol EOF token. Don't promote a partial match or fabricate a terminator the grammar didn't ask for.
  • Error-mode lexing. After SetParserState(0), lex with the most permissive mode — that's C's ERROR_STATE behavior, and recovery parity depends on it.
  • Gate every branch on validSymbols. Returning a symbol invalid in the current state is undefined-behavior territory — at best pruned, at worst an error cascade.

Before you ship

  • Run the full corpus through C and Go; diff S-expressions byte-exact — not "no errors", exact.
  • Test EOF edges: file ending at your token, in whitespace, mid-construct, empty, BOM-only.
  • Test error inputs, not just clean ones — recovery is the least-tested path in every port.
  • If stateful: Serialize → Deserialize → Serialize is a fixed point, and state fits 4096 bytes at worst nesting.
  • Any zero-width token: confirm the parser makes progress on adversarial input.

Build & deploy.

206 grammars is ~24 MB embedded. Four mutually-exclusive embedding modes let you ship anything from the whole registry to a single-language few-MB static binary.

ModeWhat you get
defaultAll 206 grammars embedded. One static binary, nothing to configure.
grammar_set_coreA curated Core100 embedded set. Narrow further at runtime with GOTREESITTER_GRAMMAR_SET=go,json,python.
grammar_subsetPick exactly the languages you ship — one grammar_subset_<lang> tag each. A single-language build drops from ~24 MB to a few MB, still self-contained.
grammar_blobs_externalNo embedded blobs; load from GOTREESITTER_GRAMMAR_BLOB_DIR at runtime (optionally mmap'd).
shell
# smallest self-contained binary: embed ONLY go + java, drop everything else
go build -tags 'grammar_subset grammar_subset_go grammar_subset_java'

# external blobs (none embedded)
go build -tags grammar_blobs_external
GOTREESITTER_GRAMMAR_BLOB_DIR=/path/to/blobs
GOTREESITTER_GRAMMAR_BLOB_MMAP=false   # disable mmap (Unix)

Releases.

The engine's road from a pure-Go proof to C-faithful error recovery. Full notes in the CHANGELOG.

v0.22.02026-07-08 · latest

Recovery-foundation checkpoint

The external lex-state election ledger for all 206 grammars, broad precise ExternalLexStates coverage, a nightly Go-vs-C perf-scan harness, and Cobol large-table recovery cleanup. Generated tables no longer silently truncate action/GOTO indexes past uint16; arena slab growth is capped against pathological parses.

v0.21.02026-07-06 · the engine release

Generalized GLR core, C-faithful recovery

A GSS-forest fast path plus C-faithful error recovery replaces the per-language approximations. Recovery for 123 elected grammars reproduces tree-sitter C's decisions — verified decision-for-decision against an instrumented v0.25.0 oracle. Real Go automatic-semicolon insertion via external scanner dropped spurious-error files 436 → 17. Full-parse memory −91% B/op; the campaign's 42-test backlog went to zero and the root suite from ~340s → ~62s.

v0.20.x2026-06

Taproot & consumer-controllable forest

Extracted the Taproot DSL harness (walk core loads a blob with no registry link), added consumer WantsForest opt-in without forking, strict-parse ErrParseStoppedEarly variants, NodeAtByte helpers, and one-pass code-understanding extractors — plus a long tail of C#/Swift real-world recovery fixes.

v0.19.x2026-05

GLR materialization & query parity

Compact/lazy final child refs survive result assembly, so queries, cursors, edits and traversal skip broad eager materialization. Nested repeated query patterns now preserve tree-sitter-compatible match rows.

v0.16.x2026-05

Native UTF-16 & grammar extensibility

UTF-16 parser/editor APIs for editor integrations, grammargen DSL constructors and extension smoke coverage, and grammar-update guardrails that block scanner-facing lock refreshes until regenerated artifacts and parity are handled.

v0.14.x2026-04

Go ships as a grammargen blob

The Go grammar became a grammargen-compiled blob from the pure-Go LR(1) state-splitting compiler, sidestepping a dead-end in tree-sitter-go's C tables. Warm-reuse heap across a six-file self-parse dropped ~54%.
